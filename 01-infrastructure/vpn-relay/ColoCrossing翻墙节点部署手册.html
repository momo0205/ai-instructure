3.1 选择 SSH 客户端

macOS 用户首选：系统自带终端不需要安装任何软件，打开「终端」App（Spotlight 搜索 Terminal），输入一行命令直接连接，最简单最安全。

3.2 macOS / Linux：系统终端连接（推荐）

  1. 打开「终端」（Spotlight 搜索 Terminal，或「应用程序 → 实用工具 → 终端」）
  2. 输入以下命令（替换为你的实际 IP）：
ssh root@你的VPS的IP地址
  1. 首次连接提示 Are you sure you want to continue connecting (yes/no)? → 输入 yes 回车
  2. 输入购买时设置的 Root Password（输入时不显示字符，这是正常的）
  3. 看到 root@xxx:~# 表示连接成功

3.3 Windows：FinalShell 连接

  1. 打开 FinalShell，左上角「新建连接」
  2. 填写：
    • 主机：你的 VPS IP 地址
    • 端口：22
    • 用户名：root
    • 密码：购买时你自己设置的 Root Password
  3. 点击「连接」，首次提示「接受密钥指纹」→ 点击「接受」
  4. 看到 root@xxx:~# 表示成功
Root Password 在哪里？就是你购买时在 ColoCrossing 配置页面「Root Password」栏自己填写的那个密码。ColoCrossing 是你自己设密码，不像其他商家需要等邮件。如果忘了，登录 cloud.colocrossing.com → 找到你的 VPS → Reset Password 重置。

四、系统初始化

连接成功后，依次复制粘贴以下命令到终端中执行（每次粘贴完一条，等执行完毕再贴下一条）。

4.1 系统更新

apt update && apt upgrade -y

大约 2-3 分钟，如果提示 "Which services should be restarted?" 直接按回车即可。

4.2 安装必要工具

apt install curl wget ufw -y

五、防火墙检查与放行 ⚠️ 关键步骤

不能跳过这一步如果防火墙没放行端口，装好面板后网页会打不开、节点连不上。ColoCrossing 默认防火墙配置不像 RackNerd 那样全开。

5.1 配置防火墙

ufw allow 22/tcp
ufw allow 2053/tcp
ufw allow 443/tcp
ufw enable

执行 ufw enable 时会提示 "Command may disrupt existing ssh connections"，输入 y 回车。放心，22 端口已经放行了，SSH 不会断。

5.2 验证防火墙

ufw status verbose

确认输出中 22、443、2053 三个端口都是 ALLOW 状态。

六、安装 3X-UI 面板

6.1 先放行 80 端口（重要）

安装脚本会询问是否签发 SSL 证书，需要 80 端口。先放行：

ufw allow 80/tcp

6.2 一键安装

bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)

脚本运行过程中会询问 SSL 证书设置，按以下回答：

问题回答说明
Choose SSL certificate setup method (default 2 for IP):直接回车选 2：Let's Encrypt for IP（6天自动续期）
Do you have an IPv6 address to include?直接回车留空跳过
Port to use for ACME HTTP-01 listener (default 80):直接回车用 80 端口

安装完成后，终端会输出如下信息（务必保存！）：

═══════════════════════════════════════════
     Panel Installation Complete!
═══════════════════════════════════════════
Username:    你的用户名
Password:    你的密码
Port:        2053
WebBasePath: 随机路径
Access URL:  http://你的IP:2053/随机路径
═══════════════════════════════════════════
立刻保存用户名和密码！这是面板的登录凭据，丢失后只能重置数据库（会丢失所有节点配置）。建议截图或记在备忘录里。

6.3 登录面板

  1. 浏览器打开 http://你的IP:2053（注意是 http 不是 https）
  2. 输入安装时显示的用户名和密码
  3. 仪表盘显示系统信息 → 安装成功！
打不开？确认输入的是 http:// 不是 https://。如果还打不开，执行 ufw status verbose 检查 2053 端口是否是 ALLOW 状态。

6.4 忘记密码怎么办

SSH 登录后执行以下命令重置（替换为你想要的用户名和密码）：

systemctl stop x-ui
rm /etc/x-ui/x-ui.db
systemctl start x-ui

执行后用 admin / admin 登录，进去后立即改密码。

七、创建 VLESS + Reality 节点 🔥 最关键的一步

理解「入站」和「客户端」的关系入站 = 服务器开的一扇门（协议、端口、加密方式）。客户端 = 一把钥匙（UUID 凭证）。必须先建门，再配钥匙，才能导出连接链接。

7.1 添加入站

  1. 左侧菜单 → 入站 → 右上角 + 添加入站
  2. 表单分为四个区域（基础配置 / 协议 / 传输 / 安全），按下表逐项填写：

基础配置区域

配置项填写值说明
备注ColoCrossing-LA随意命名，便于识别
协议vless下拉选择 vless
端口443Reality 使用 443

协议区域

配置项填写值说明
加密None默认，不用改
Vision testseed保持默认（900 500 900 256）不用改，这是 xtls-rprx-vision 的流控参数

传输区域

配置项填写值说明
传输RAW（默认）RAW 即 TCP，不用改

安全区域（选 Reality 后展开）

配置项填写值说明
安全Reality⚠️ 必须选 Reality，不是 TLS
uTLSchrome已默认，不用改

填完后点页面底部 「添加」 按钮保存。

7.2 添加客户端（必须！）

⚠️ 不添加客户端，导出的链接是空的！入站创建完后，必须为它添加至少一个客户端（连接凭证），才能生成可用的导入链接。
  1. 左侧菜单 → 客户端 → + 添加客户端
  2. 按下表填写：
配置项填写值说明
邮箱自动生成，不用改仅用于面板内识别
UUID自动生成，不用改连接凭证，唯一标识
Flowxtls-rprx-vision⚠️ 必须选这个
过期时间留空留空 = 永不过期
总流量 (GB)00 = 不限流量
关联入站ColoCrossing-LA关联到刚创建的入站

点击 「创建」 保存。

多设备怎么办？一个客户端可以同时在多台设备上使用（就像 WiFi 密码多台设备共用）。如果想单独管理每台设备的流量，可以为每台设备创建一个独立客户端。

7.3 导出节点链接

  1. 左侧菜单 → 入站
  2. 找到 ColoCrossing-LA 那行，点右边的 ⋮（三个点）
  3. 选择 「导出链接」
  4. 弹窗中会出现 vless://... 开头的链接
  5. 点击 「复制」 按钮复制链接
Reality 协议是什么？目前抗封锁能力最强的协议，把代理流量伪装成访问微软的正常 HTTPS 流量，极大降低被检测和封锁的概率。

八、开启 BBR 加速

BBR 是 Google 开发的 TCP 拥塞控制算法，可以显著提升跨境网络速度，开启后有 20-50% 的速度提升。

8.1 一键开启

echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
sysctl -p

8.2 验证

sysctl net.ipv4.tcp_congestion_control

输出应为：net.ipv4.tcp_congestion_control = bbr

十、故障排查

10.1 节点连不上

检查项命令/操作正常结果
服务是否运行systemctl status x-uiactive (running)
443 端口是否监听ss -tlnp | grep 443有输出
防火墙是否放行ufw status verbose | grep 443ALLOW

如果服务没运行：

systemctl restart x-ui

10.2 面板打不开

  1. 确认输入的是 http:// 不是 https://
  2. SSH 登录后执行：systemctl restart x-ui
  3. 检查防火墙：ufw status verbose | grep 2053
  4. 如没有 ALLOW，回第五步重新配置防火墙

10.3 速度慢

  1. 确认 BBR 已开启：sysctl net.ipv4.tcp_congestion_control
  2. 晚高峰（20:00-23:00）普通 BGP 线路必然降速，这是正常的
  3. 如果任何时候都慢，可能 IP 段质量不好，考虑发工单要求更换

10.4 面板常用管理命令

x-ui              # 进入管理菜单
x-ui restart      # 重启面板
x-ui status       # 查看面板状态
x-ui update       # 更新面板
x-ui log          # 查看日志

10.5 更换 IP（如被墙）

如果 IP 被墙了（ping 不通、端口被封）：

  1. 登录 cloud.colocrossing.com
  2. 找到你的 VPS → 查看是否有 IP 更换选项
  3. 如果没有自助更换选项，提交一个工单（Support Ticket），说明需要更换 IP
  4. 拿到新 IP 后，SSH 登录，更新 3X-UI 面板中的节点 IP 即可
