七、开启 BBR 加速（可选但推荐）

BBR 是 Google 开发的 TCP 拥塞控制算法，可显著提升 VPS 的网络吞吐量，尤其在跨境高延迟场景下效果明显。

7.1 一键开启（推荐）

echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
sysctl -p

7.2 验证 BBR 是否已开启

sysctl net.ipv4.tcp_congestion_control
# 输出应为：net.ipv4.tcp_congestion_control = bbr

lsmod | grep bbr
# 输出应包含：tcp_bbr

7.3 进阶优化（可选）

如果需要更激进的加速，可以安装 BBRv3：

# BBRv3 一键脚本（社区维护）
wget -O tcpx.sh "https://github.com/ylx2016/Linux-NetSpeed/raw/master/tcpx.sh" && chmod +x tcpx.sh && ./tcpx.sh
注意BBRv3 需要较新内核，建议先在原版 BBR 上观察效果，满意即可，不必折腾 BBRv3。

9.2 常见故障排查

故障 1：节点突然连不上了

排查步骤：

  1. Ping 一下你的 VPS IP → 如果 ping 不通，IP 可能被墙了
  2. 如果能 ping 通但 443 端口不通 → 端口被封了（常见）
  3. 如果能 ping 通且端口通，但节点不通 → 检查 Xray 进程：systemctl status x-ui

IP 被墙的解决方案：

  • RackNerd：先换端口试试（如 8443），如果 IP 被封 → 花 $3 购买新 IP
  • 搬瓦工：在 KiwiVM 面板切换到其他机房（免费换机房=免费换 IP）

故障 2：速度很慢

  1. 是否在晚高峰（20:00-23:00）？普通 BGP 线路晚高峰必然降速
  2. 检查 BBR 是否开启：sysctl net.ipv4.tcp_congestion_control
  3. 搬瓦工用户：尝试切换到 DC6 或软银机房

故障 3：面板网站打不开

  1. SSH 登录后执行：systemctl status x-ui
  2. 如果服务挂了：systemctl restart x-ui
  3. 检查防火墙：iptables -L -n | grep 2053
