# 覆盖检查清单

生成测试用例后，使用此清单自检覆盖完整性。

## 必须覆盖

### 显示条件

- [ ] 每个组件的显示条件已覆盖
- [ ] 每个组件的隐藏条件已覆盖
- [ ] 组件间的位置关系已说明

### 状态表现（四维度）

- [ ] 每个状态的 **文案** 已覆盖
- [ ] 每个状态的 **视觉效果** 已覆盖（动效、样式）
- [ ] 每个状态的 **点击行为** 已覆盖
- [ ] 每个状态的 **hover 行为** 已覆盖
  - [ ] Tooltip 内容
  - [ ] 鼠标指针样式（小手/箭头）

### 状态流转

- [ ] 正常流程的状态流转已覆盖
- [ ] 异常情况的状态回退已覆盖
- [ ] 失败后的恢复/重试路径已覆盖

### 多语言

- [ ] 每个状态的英文文案已覆盖
- [ ] Tooltip 的英文内容已覆盖
- [ ] 通知的英文内容已覆盖（如有）

## 按需覆盖

根据具体场景选择性覆盖：

### 页面生命周期

- [ ] 页面刷新后状态恢复
- [ ] 组件卸载时资源清理
- [ ] 组件重新挂载时初始化

### 上下文切换

- [ ] 切换上下文后状态正确切换
- [ ] 多上下文状态隔离

### 多组件联动

- [ ] 组件 A 操作时，组件 B 状态同步
- [ ] 所有联动组件操作完成后同步恢复

### 并发/边界

- [ ] 快速连续点击处理（防抖）
- [ ] 同时触发多个操作（并发冲突）
- [ ] 空数据场景
- [ ] 大数据量场景

### 事件处理

- [ ] 事件冒泡处理（嵌套组件）
- [ ] 禁用状态下的事件屏蔽

### 错误处理

- [ ] 网络错误场景
- [ ] 超时场景
- [ ] 服务不可用场景
- [ ] 错误兜底页展示

## 检查流程

```
1. 生成测试用例
       │
       ▼
2. 对照「必须覆盖」逐项检查
       │
       ├─ 有遗漏 → 补充用例
       │
       ▼
3. 根据具体场景，选择「按需覆盖」项
       │
       ├─ 适用 → 补充用例
       │
       ▼
4. 检查完成，输出最终用例
```

## 常见遗漏点

| 遗漏类型 | 说明 | 检查方法 |
|---------|------|---------|
| hover 行为 | 只写了点击，忘了 hover | 每个可交互元素都检查 |
| 鼠标指针 | 忘了 hover 时指针样式 | 每个 hover 都检查指针 |
| 禁用状态 | 只写了正常点击 | 每个有禁用态的元素都检查 |
| 异常回退 | 只写了正常流程 | 每个流程都考虑失败场景 |
| 刷新恢复 | 忘了页面刷新后的状态 | 每个有状态的组件都检查 |
| 多语言 | 忘了英文文案 | 最后单独过一遍 |

## 用例数量参考

| 组件复杂度 | 状态数 | 预期用例数 |
|-----------|--------|-----------|
| 简单 | 1-2 | 5-15 条 |
| 中等 | 3-4 | 15-30 条 |
| 复杂 | 5+ | 30+ 条 |

**注意**：用例数量不是越多越好，关键是覆盖完整、无重复。
