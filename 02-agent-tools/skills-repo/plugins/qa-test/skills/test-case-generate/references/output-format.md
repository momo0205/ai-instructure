# 输出格式规范

## Checkbox 格式

所有测试用例必须使用 `- [ ]` 格式，方便在 Markdown 中直接勾选确认。

```markdown
- [ ] 1.1 [条件] 时，显示 [组件]
- [ ] 1.2 [条件] 时，不显示 [组件]
```

**注意**：不要使用表格格式，因为表格中的 Checkbox 无法交互。

## 编号规则

| 编号范围 | 用途 | 说明 |
|---------|------|------|
| 0.x | 场景梳理 | 端到端流程测试 |
| 1.x ~ N.x | 功能点测试 | 按分组编号 |

## 场景梳理格式

使用 `→` 箭头串联操作步骤，开头用「场景名」标注：

```markdown
### 场景梳理

- [ ] 0.1 「正常流程」用户执行操作→系统响应→显示结果
- [ ] 0.2 「异常处理」用户执行操作→发生错误→显示错误提示→可重试
- [ ] 0.3 「边界场景」特定条件→执行操作→验证边界行为
```

**场景梳理要点**：
- 覆盖正常流程
- 覆盖异常流程
- 覆盖边界场景
- 每个场景是完整的用户旅程

## 功能点测试格式

按分组组织，每条用例描述单个测试点：

```markdown
### ▼ 1. 显示条件

- [ ] 1.1 [条件A] 时，显示 [组件]
- [ ] 1.2 [条件B] 时，不显示 [组件]
- [ ] 1.3 [组件] 位于 [参照物] 的 [位置]

### ▼ 2. [状态名] 状态

- [ ] 2.1 [元素] 文案显示「XX」
- [ ] 2.2 hover 时显示 Tooltip「XX」
- [ ] 2.3 hover 时鼠标显示小手指针
- [ ] 2.4 点击后，触发 [行为]
```

## 文案粒度

### 具体文案（推荐）

```markdown
- [ ] 2.1 按钮文案显示「提交」
- [ ] 2.2 hover 时显示 Tooltip「点击提交表单」
```

### 模糊描述

```markdown
- [ ] 2.1 按钮显示正确的文案
- [ ] 2.2 hover 时显示功能说明 Tooltip
```

**选择建议**：
- QA 验收测试 → 具体文案（便于验证）
- 通用测试框架 → 模糊描述（更灵活）

## 多语言格式

### 方式 A：单独分组（推荐）

```markdown
### ▼ N. 多语言

- [ ] N.1 切换英文后，[状态A] 显示「English Text」
- [ ] N.2 切换英文后，[状态B] 显示「Another Text」
- [ ] N.3 切换英文后，Tooltip 显示「English tooltip」
```

### 方式 B：每条用例双语

```markdown
- [ ] 2.1 按钮文案显示「提交」/「Submit」
- [ ] 2.2 hover Tooltip 显示「点击提交」/「Click to submit」
```

## 完整文档结构

```markdown
## 📋 [区域名] - 测试用例

### 场景梳理

- [ ] 0.1 「场景名」流程描述...
- [ ] 0.2 「场景名」流程描述...

### ▼ 1. 显示条件

- [ ] 1.1 用例描述
- [ ] 1.2 用例描述

### ▼ 2. [状态A] 状态

- [ ] 2.1 文案...
- [ ] 2.2 视觉...
- [ ] 2.3 点击...
- [ ] 2.4 hover...

### ▼ 3. [状态B] 状态

- [ ] 3.1 文案...
- [ ] 3.2 视觉...
- [ ] 3.3 点击...
- [ ] 3.4 hover...

### ▼ 4. 状态流转

- [ ] 4.1 正常流转...
- [ ] 4.2 异常回退...
- [ ] 4.3 刷新恢复...

### ▼ 5. [其他分组]

- [ ] 5.1 ...

### ▼ N. 多语言

- [ ] N.1 英文文案...
- [ ] N.2 英文 Tooltip...
```
