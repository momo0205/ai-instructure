## 一、背景

为确保 Agent 和用户在使用仓库时有一致的体验，需要对脚本设计和 Agent 交互行为制定规范。

## 二、脚本设计规范

### setup.sh 功能与流程

`scripts/setup.sh` 是环境初始化脚本，遵循以下流程：

1. **环境检测**：检查 Git Hook 是否已正确配置
2. **检测成功**：返回成功，直接继续执行
3. **检测失败**：进入初始化设置流程，自动配置
4. **设置结果**：
   - 成功：输出成功信息，继续执行
   - 失败：抛出非零错误码，输出详细错误原因

### 错误处理要求

脚本必须遵循以下错误处理规范：

1. **详细错误信息**：设置失败时，必须输出：
   - 错误原因
   - 错误细节
   - 解决建议

2. **错误码机制**：
   - 成功：exit 0
   - 失败：exit 非零值，并输出具体错误

3. **执行反馈**：每个步骤执行前后必须有明确的打印信息，让用户了解进度

## 三、Agent 交互规范

### 脚本调用规范

智能体在调用任何脚本或命令前，必须：

1. **说明原因**：向用户说明为什么需要调用此命令
2. **展示命令**：让用户知道即将执行的完整命令
3. **等待确认**：对于 setup.sh 等关键操作，建议让用户确认后再执行

### 示例

```
我需要先运行 setup.sh 来验证和配置环境。这将检查 Git Hook 是否已设置，如果未设置则自动配置。

运行命令：source scripts/setup.sh

是否继续？
```
