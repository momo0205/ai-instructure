📦 前置步骤（所有工具通用）

步骤 1：安装 Python 依赖

a-stock-data 需要 4 个 Python 包。Mira 的 Python 工具（tools/mira_data/）是自包含的，无额外依赖。

pip install mootdx requests pandas stockstats
💡 mootdx 需要在国内网络环境运行（通达信 TCP 7709 端口，境外 IP 可能连不上）。其他包无此限制。

步骤 2：确认仓库位置

两个仓库应在此目录下：

ai/04-applications/investment-research/
├── a-stock-data/
│   ├── SKILL.md ← 2119 行，内嵌 27 个数据端点的 Python 代码
│   ├── README.md
│   └── CHANGELOG.md
└── Mira/
    ├── CLAUDE.md ← Claude Code 自动加载
    ├── AGENTS.md ← Codex 自动加载
    ├── MIRA.md ← 唤醒词定义
    ├── OPERATING_CONTRACT.md ← 懒加载地图
    ├── START_HERE.md ← 用户入口
    ├── AGENT_QUICKSTART.md ← Agent 执行规则

    ├── skills/ ← 9 个分析技能

    ├── templates/ ← 研究包模板

    ├── tools/mira_data/ ← Python 数据适配器

🧡 Claude Code（推荐首选）

兼容状态：✅ 原生支持，可直接使用

步骤 1：安装 Claude Code

npm install -g @anthropic-ai/claude-code

# 启动并登录
claude login

按提示完成 Anthropic 账号登录和付费订阅（推荐 Pro，$20/月）。

步骤 2：安装 a-stock-data Skill

Claude Code 会自动加载 ~/.claude/skills/*/SKILL.md 下的所有 skill 文件。

# 创建目录
mkdir -p ~/.claude/skills/a-stock-data

# 复制 SKILL.md（或软链接）
cp /path/to/ai/04-applications/investment-research/a-stock-data/SKILL.md \
   ~/.claude/skills/a-stock-data/SKILL.md
📝 Claude Code 启动时会扫描此目录。SKILL.md 中的 yaml frontmatter（name: a-stock-data）告诉 Claude 这个 skill 的名字和用途。

步骤 3：启动 Mira 研究会话

在 Mira 仓库目录下启动 Claude Code：

cd /path/to/ai/04-applications/investment-research/Mira
claude

Claude Code 自动读取根目录的 CLAUDE.md（37 行），其中定义了：

  • 先读 AGENTS.md
  • 再读 MIRA.md（唤醒词定义）
  • 再读 OPERATING_CONTRACT.md（懒加载地图）
  • 用户说 Mira 时进入 Mira Mode

步骤 4：第一个研究任务

测试用例 1：美股快速浏览（不依赖 a-stock-data，验证 Mira 协议本身）

Mira, 看一下 AAPL
我真正想判断的是：iOS 19 周期是否能驱动 iPhone 换机潮。
市场范围：美股
时间边界：未来 2 个 quarter
输出深度：quick_map

预期行为：Claude Code 读 START_HERE.md → 路由到 analysis-routing.md → 判断 quick_map → 搜索公开数据 → 输出 source notes + refresh triggers。产出物暂存在对话中，不会写入文件（quick_map 是非正式 triage）。

测试用例 2：A 股研究（依赖 a-stock-data）

Mira, 研究一下 600519（贵州茅台）
市场范围：A 股
时间边界：最新季度
输出深度：standard
输出要求：research package，包含 evidence log

预期行为：Claude Code 检测到 A 股 → 触发 a-stock-data skill → 用 mootdx 取 K 线/财务/F10 + 腾讯取 PE/PB/市值 + 东财取北向资金/龙虎榜 → 写入 cases/600519-2026-06/，包含 investment-memo.md + evidence-log.csv。

步骤 5：非交互式自动化（在 MacBook 服务器上）

Claude Code 支持 -p 参数进行单次非交互式查询：

# 在 MacBook 上的 cron job 示例
cd /path/to/Mira && claude -p "Mira, 做一份 A 股收盘简报，市场范围：沪深两市，输出 standard market briefing package" --output-format stream-json > /tmp/mira-briefing-$(date +%F).json

可以把这条命令加入 cron，每天 15:30（收盘后）自动执行，产出物保存到 cases/ 目录。

💙 Codex（OpenAI 官方）

兼容状态：✅ 官方支持，文档已验证

步骤 1：安装 Codex

npm install -g @openai/codex

# 登录
codex login

按提示完成 OpenAI 账号登录。

步骤 2：无需额外配置

Codex 自动读取项目根目录的 AGENTS.md。Mira 的 AGENTS.md（80 行）定义了完整的读序和执行规则。启动 Codex 时直接在 Mira 目录下：

cd /path/to/ai/04-applications/investment-research/Mira
codex

Codex 自动加载 AGENTS.md → 按读序加载 MIRA.md → OPERATING_CONTRACT.md。

步骤 3：a-stock-data 的集成

Codex 不自动加载 ~/.claude/skills/ 目录。需要让 Codex 主动读取 a-stock-data 的 SKILL.md。最简单的方式：

# 在 Codex 会话中第一次使用时说：
请先读取 ../a-stock-data/SKILL.md，将其作为 A 股数据技能加载。
之后任何涉及 A 股的研究任务，优先从这个文件获取数据端点。

或者将 a-stock-data 的 SKILL.md 复制到 Mira 目录下，改名为 skills/a-stock-data.md，然后在 AGENTS.md 中添加一行引用。但这不是官方做法，属于 hack。

⚠️ Codex 对 a-stock-data 的支持不如 Claude Code 原生。如果以 A 股研究为主要目标，建议用 Claude Code。

# A 股测试（验证 a-stock-data 集成）
先加载 ../a-stock-data/SKILL.md 作为 A 股数据源。

🐳 CodeWhale（DeepSeek 终端 Agent）

兼容状态：⚠️ 未验证，需测试。Python 执行能力有，但协议跟随能力未知。

已知信息

  • CodeWhale 是终端编码 Agent，可以读文件、执行 Python、编辑文件
  • 不确定：能否像 Claude Code/Codex 一样严格遵循一个多层 Markdown 协议
  • 不确定：面对 Mira 的 OPERATING_CONTRACT.md 懒加载地图，能否先读 maps 再按需加载 loop/skill/template
  • 不确定：能否按 evidence-log.csv schema 写出格式正确的研究包
📝 建议花 10 分钟在主力机上做一次快速测试（见步骤 3），不需要做任何配置。测试失败就放弃，不浪费时间。

步骤 1：安装 CodeWhale

npm install -g codewhale

# 设置 DeepSeek API Key
echo 'export DEEPSEEK_API_KEY="sk-..."' >> ~/.zshrc
source ~/.zshrc

# 启动
codewhale

步骤 2：进入项目目录并手动加载协议

CodeWhale 不会自动加载 CLAUDE.md 或 AGENTS.md。需要在会话开始时手动指令：

cd /path/to/ai/04-applications/investment-research/Mira
codewhale

启动后，先输入以下引导指令：

你现在在 Mira 研究协议的仓库中。请先按顺序读取以下文件来理解项目规则：
1. MIRA.md（唤醒词定义和身份边界）
2. OPERATING_CONTRACT.md（懒加载地图和核心契约）
3. START_HERE.md（用户入口）

读完后告诉我你理解了哪些规则，然后我会给你研究任务。
💡 关键观察点：CodeWhale 读完后是照做（先读完指定的文件再回复），还是自由发挥（随便回复几句"懂了"但其实没读）？这是判断能否用下去的核心指标。

步骤 3：快速验证测试（10 分钟）

Round 1：简单美股路由

预期：CodeWhale 应该做以下事情——
① 读 loops/analysis-routing.md（如果它之前没读）
② 判断 depth = quick_map
③ 搜索 AAPL 的公开信息（或调 Yahoo Finance API）

Pass 条件：输出至少包含"stale_after"或"must_refresh_if"字段，且能看到对数据来源的标注。

Round 2（仅 Round 1 通过后）：A 股数据

预期：CodeWhale 从 SKILL.md 中找到 mootdx / 腾讯 / 东财的 Python 代码段，执行取数据，按 Mira 模板写研究包。

💜 CodeFlicker（快手内部 Agent）

兼容状态：⚠️ 未验证。SKILL.md yaml 格式匹配，但协议跟随能力未知。

已知信息

  • a-stock-data 的 SKILL.md 使用 yaml frontmatter（---\nname: a-stock-data\n...），这和 CodeFlicker 的 skill 格式完全一致
  • Mira 没有 SKILL.md 格式的文件——它用的是 CLAUDE.md / AGENTS.md / OPERATING_CONTRACT.md 这套协议系统
  • 不确定：CodeFlicker 能否在打开一个目录后，自动或手动加载 CLAUDE.md / AGENTS.md 作为项目规则
  • 不确定：即使加载了，能否遵循 OPERATING_CONTRACT.md 规定的懒加载模式

步骤 1：安装 a-stock-data Skill

将 a-stock-data 的 SKILL.md 复制到 CodeFlicker 的 skills 目录：

# 方式 A：复制到 CodeFlicker 项目级 skills 目录（如果有的话）
cp ../a-stock-data/SKILL.md /path/to/.codeflicker/skills/a-stock-data/SKILL.md

# 方式 B：直接在当前会话中加载
# 在 CodeFlicker 中说：
# 请读取 /path/to/a-stock-data/SKILL.md，这是一个 A 股数据工具包。

步骤 2：加载 Mira 协议（手动）

CodeFlicker 没有自动读取 CLAUDE.md 的机制（当前会话就是 CodeFlicker，我能读文件但你无法让我自动进入"Mira Mode"——需要你主动指令）：

请以 Mira 研究协议的方式工作。先读取以下文件：
1. Mira/MIRA.md（身份定义）
2. Mira/OPERATING_CONTRACT.md（懒加载地图）
3. Mira/START_HERE.md（用户入口）

读完后，按照这些协议规则处理后续的所有研究任务。

步骤 3：快速验证测试

加载协议后，测试一个简单任务：

Mira, 看一下 AAPL。市场范围：美股。输出深度：quick_map。

鉴定标准和 CodeWhale 相同：看输出是否遵守了 Mira 的 evidence/stale_after/follow-up 三件套规则。

⚠️ CodeFlicker 的适用性取决于它读取多层 Markdown 指令后能否严格遵循。这是一个架构级问题，不是简单测试能确认的——Mira 的 OPERATING_CONTRACT.md 要求 Agent"懒加载"（先读 maps，再按需加载具体 loop/skill），这需要 Agent 理解"先看地图再前进"的元指令。

🔧 Mira 内置 Python 工具（可选）

Mira 的数据适配器和验证脚本

Mira 的 tools/mira_data/ 提供美股数据获取功能，AI Agent 不需要手动调用（Agent 会在需要时自己调用）。但作为使用者，你可以手动验证：

# 查看数据适配器列表
ls Mira/tools/mira_data/adapters/
# 输出：bls.py  ibkr_gateway.py  sec_companyfacts.py  yahoo_chart.py

# 手动运行 repo 验证（检查研究包格式合规）
cd Mira
python3 scripts/validate_repo.py cases/aapl-2026-04/
# 输出：case-format-ok 或 列出格式错误

# 生成 case manifest
python3 scripts/generate_case_manifests.py cases/aapl-2026-04/
# 在 case 目录下生成 research-package-manifest.json
